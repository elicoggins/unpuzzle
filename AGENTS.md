# Unpuzzle — App Overview

## What It Is

Chess puzzle apps suffer a common problem. Users know they're grinding puzzles, so they calculate tactics and sacrifices far more willingly than they would in a standard timed game.

This app addresses that issue by presenting an unmarked position. Sometimes, it will be a puzzle with a tactic present. Sometimes, it might be a sharp position with a couple winning moves. Sometimes it will be completely balanced and a simple developing move is advisable. 

Study the position, determine what you think is a good move. Stockfish evaluates your choice and scores it against the best move in the position.

## Tech Stack

- **Framework**: Next.js (App Router, static export via `output: "export"`)
- **Hosting**: GitHub Pages at `https://elicoggins.github.io/unpuzzle/` — deployed via GitHub Actions on push to `main`
- **Language**: TypeScript
- **Styling**: Tailwind CSS v4, CSS variables for theming
- **Chess logic**: `chess.js` for move generation/validation
- **Board rendering**: `react-chessboard` v5 (custom pieces — see user memory for bundling quirks)
- **Drag/drop**: dnd-kit (internal to react-chessboard), `dragActivationDistance: 0`
- **Engine**: Stockfish 18 lite single-thread WASM, served from `public/stockfish/`
- **Animation**: framer-motion
- **Package manager**: pnpm

## Game Flow (`/play`)

1. **Loading** — a position is randomly weighted-sampled from `src/lib/positions/`
2. **Playing** — user makes a move (click-to-move or drag). Right-click draws arrows; right-click on a square clears selection and hides move dots. Capture clicks are handled in `onSquareMouseDown` (not `onClick`) to work around dnd-kit eating click events.
3. **Confirming** — move shown with confirm/undo. Clicking the board also undoes.
4. **Evaluating** — Stockfish analyzes the original position (multi-PV for engine lines) and post-move position. Depth progress bar shown.
5. **Scored** — `ScoreReveal` animates in with: headline + glyph circle, pawn loss / accuracy metrics, best move, category. Engine lines stay visible above.

## Eval Pipeline

All evals from Stockfish are **side-to-move perspective** (positive = side to move is winning). The code consistently converts to **white perspective** (positive = white winning) for display and bar rendering.

- **CPL formula**: `max(0, evalAfterBestCp - evalAfterFromOrigPerspective)` — compares post-best vs post-played from the original side's perspective. Eliminates ply-bounce.
- **Accuracy**: win-probability sigmoid `100 / (1 + e^(-0.00368208 * cp))`. Compares `evalAfterBest` vs `evalAfterPlayed`. A 50pp win-probability drop → 0% accuracy. Uses `evalAfterBest` as the reference (not `evalBefore`).
- **`isMateAfterPlayed`** is set to `false` when `afterGame.isCheckmate()` is true — delivering mate is not a blunder.

## Position Pool

- **Source**: `src/lib/positions/` — one file per category, generated by `scripts/fetch-positions.mjs`, **do not hand-edit**
- **Index**: `src/lib/positions/index.ts` — exports `CuratedPosition` type, `getRandomPosition()`, weighted picker logic
- **Category files**: `tactical.ts`, `balanced.ts`, `critical.ts`, `tricky.ts`, `endgame.ts`
- **Categories**: `tactical`, `balanced`, `critical`, `tricky`, `endgame`
- **Count**: ~910 positions (tactical:60, balanced:610, critical:76, tricky:60, endgame:106); no hardcoded FENs — all fetched from Lichess API at generation time
- **Weights**: `tactical:0.20, balanced:0.25, critical:0.20, tricky:0.15, endgame:0.20`
- To regenerate: `node scripts/fetch-positions.mjs` (writes directly to `src/lib/positions/*.ts`)

## Layout

Three-column layout on `/play`:

```
[Left Panel 220px] [Eval Bar + Board] [Right Panel 260px]
```

**Left panel** (top to bottom): puzzle info (move number, phase, side to move) → session stats (puzzles, ACPL, streak/best streak toggle) → engine status (while loading) → move history box (scrollable, browse mode after scoring)

**Right panel** (top to bottom): move history → engine lines (3 PV rows, always rendered) → feedback panel (state machine: playing / confirming / evaluating / scored+next button)

**Eval bar**: 28px wide, fills white from the side of the board where white sits. For black orientation, fills from the top.

## Settings

- **Accent color**: 3 presets (Green, Gold, Pink) + custom hex color picker; stored in `localStorage` as `accent-color`
- **Heading font**: 4 options — Orbitron, Space Grotesk, Audiowide, Press Start 2P; stored in `localStorage` as `heading-font`
- **Engine depth**: 18 / 20 / 24 (default 18); stored in `localStorage` as `engine-depth`

## Known Quirks

- react-chessboard v5 pre-built bundle breaks under Turbopack/Webpack (see user memory). Use `--webpack` flag and custom pieces.
- `dragActivationDistance: 0` means dnd-kit activates drag on `pointerdown`, eating the subsequent `click` event. Capture moves must be executed in `onSquareMouseDown`.
- All Stockfish evals are side-to-move perspective; always negate after a move when converting to original-side perspective.
